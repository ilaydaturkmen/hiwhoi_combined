IP , PACKETS & ROUTING

How is it possible for every piece fo data to be delivered to you reliably? IP address also known as Internet Protocol, is an address of your network hardware. It is a series of numbers which identify computers on the Internet. and helps getting the message from the server and delivering it to you. IP address is made up of numbers or characters. For example - 516.487.14.562 All devices that have access to the internet have a unique IP address. IP address represents where you live on the Internet and are needed to send messages to outside of your network.

An IP address has two parts:

  • network ID
  • host ID

WHAT'S DNS?

DNS ,known as Domain Name System, validates all Internet traffic, and blocks illegally formed DNS packets. Without DNS the Internet as we know it today, would collapse, and we would all be licking stamps to pay our bills, reading the newspaper to see what movies were showing, and driving to an actual store to purchase something. Computers that make up the Internet set up in large networks that communicate with each other via underground, wires and are identified using string of numbers known as IP addresses. DNS is used to translate actual name into these numbers. When you enter a URL, for example www.webdesign.com into your address bar. There’s actually a dot at the end that we don’t realize which represents the root of the Internet name’s space. Browser first determine the IP address. What if it’s an unknown iP address? The operating system is configured to ask resolving name server, for IP addresses is does not know. This resolving name server is the workhorse of the DNS lookup. It’s either configured manually or automatically within your operating system. Your operating system asks, the resolving name serve for the website.

HTTP & HTTPS PROTOCOLS

    HTTP follows the standard client-server model or request response model. You’re request something and get a reply back.

    The HTTPS (hypertext transfer protocol secure) in web-browser indicates that your communication with the server is encrypted meaning security of data transfer

    The big difference between HTTP and HTTPS is that one is secure and one is unsecure.

    • HTTP - unsecure
    • HTTPS - secure

    A browser sends an HTTP request to the web

    Web server receives the request

    Server returns an HTTP response to the browser

    Client receives the response and runs the website

    HTML AND CSS

    Every webpage on the Internet uses HTML, CSS, and JavaScript. They are the foundational coding languages of the Internet. Web Development is what you see when you go to a website.

    • HTML -Hyper Text Language, the builder
    • CSS - Cascading Style Sheets, the artist
    • HTML and CSS make up what is called “Front End Web Development.” Front en development is what you see when you land on a page, as opposed to Back End Development, is actually what makes an application on the web work.
    • Web Browsers translate these languages into the visual webpage.

    HTML essentially would define all the content, the text, the images, the links, etc. On the other hand, CSS designs the HTML. CSS can’t live without HTML or else would be nothing to style. It’s responsible for outlining the colors, the fonts, and the positioning of content.

    • HTML is for adding meaning to raw content by marking it up.
    • CSS is for formatting that marked up content.
